Data Flow:
1. Browser calls: GET /api/clusters
2. Server receives request
3. Server calls: supabase.from('clusters').select(...)
4. Database returns: cluster rows
5. Server transforms data → JSON
6. Browser receives: JSON with cluster data
7. Frontend renders: Cluster cards in left panel

DATABASE SCHEMA:
================

Tables created/expected:
1. clusters
   - cluster_id (TEXT, PRIMARY KEY)
   - summary (TEXT)
   - email_count (INTEGER)
   - created_at, updated_at (TIMESTAMPTZ)

2. emails
   - message_id (TEXT, PRIMARY KEY)
   - sender, sender_email (TEXT)
   - subject, body, body_html (TEXT)
   - status, is_read, is_deleted (status/BOOLEAN)
   - created_at, updated_at (TIMESTAMPTZ)

3. email_clusters (junction table)
   - id (BIGSERIAL, PRIMARY KEY)
   - cluster_id (TEXT, FOREIGN KEY)
   - message_id (TEXT, FOREIGN KEY)
   - UNIQUE(cluster_id, message_id)

4. email_ingestion_logs
   - id, message_id, source, status, details
   - created_at

5. email_processing_errors
   - id, message_id, error_message, failed_stage, timestamp
